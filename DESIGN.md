# Design: Red Pencil
**Date:** 2026-06-06 · **Status:** confirmed
**Archetype:** Creator (Sage inflection) · **Register:** brand
**DNA:** Neo-Brutalist + type voice from Editorial Minimalism · **Dominant axis:** layout discipline

## Direction
The proofreader's desk. Markdown comes in wrong, gets marked up in red, leaves correct — the site stages that correction as physical paper on a desk. Neo-brutalist bones (hard borders, offset shadows, sheets that rotate) carry the README's dry wit ("Your team sees a mess."), while Fraunces Black display type supplies the craftsman's letterpress credibility. Audience is Claude Code users; the page must itself be proof that penman's typographic obsession is real — every px value shown is the actual shipping value from `src/tokens.js`.

## Signature move
The hero before/after is two physical sheets of paper. The "before" sheet is rotated −1.2° and shows a real broken paste, with red proof-strikes (accent-9 strokes) through the worst failures and a red Fraunces-italic margin note. The "after" sheet sits dead straight at 0°, shows the penman output, and carries the page's ONLY cyan element: a press stamp ("set by penman"). On hover the after-sheet translates −2px,−2px and its offset shadow grows 6px→8px — paper being slid into place. 150ms, sharp ease-out.

## Type
- Display: Fraunces (Google Fonts; weights 600 italic, 900) — fallback: Georgia, serif
- Body: Archivo (Google Fonts; 400, 500, 600) — fallback: system-ui, sans-serif
- Mono: IBM Plex Mono (Google Fonts; 400, 600) — code, commands, recipe values — fallback: ui-monospace, monospace
- Scale: 1.5 (perfect fifth) from 16px — 16 / 24 / 36 / 54 / 81 / 122. Display steps fluid: e.g. hero `clamp(54px, 9vw, 122px)`
- Leading: body 1.5 · display 1.05 · Weights: display 900 only (Fraunces Black), 600 italic for margin notes; body 400/500; mono 400, 600 for emphasis

Color roles:
- Paper: `--background` / `--surface` (sheets). Ink: `--text` (#312d2c — never pure black).
- Red (`--accent-9/-10/-11`): proof-strikes, margin notes, the CTA, current-nav. Marks and corrections ONLY — never body text, never section backgrounds.
- Cyan (`--cyan-9/-11`): the "after/fixed" press stamp and success states. Nowhere else.
- Light scheme is canonical (paper IS the metaphor) and is the ONLY active scheme in v1 — dark tokens are recorded above for a future pass, gated on the desk-at-night art direction (see Open questions). No `prefers-color-scheme` switch until then.

## Space, shape, depth
- Spacing scale: 4 / 8 / 16 / 24 / 40 / 64 / 104px — section gaps use the top steps; rhythm varies (no uniform 24px everywhere)
- Radius: 0. Everywhere. No exceptions.
- Borders: 2px solid var(--text) on sheets/panels; 3px on the hero sheets; 1px var(--border) hairlines inside tables
- Shadows: hard offset only, zero blur — `6px 6px 0 var(--text)` resting, `8px 8px 0` hover. Never pure #000 (ink #312d2c is the shadow color). No soft/blurred shadows anywhere
- Sheets may rotate ±1.2° max; rotation applies only to "paper" elements, never to running text

## Motion
- Timing: 100ms micro / 150ms standard / 200ms max per step · Easing: cubic-bezier(0.2, 0, 0, 1) — sharp ease-out
- Allowed: translate on hover/press (paper slides), shadow offset growth, stamp scale-in once on first scroll into view
- **Hero entrance choreography (added 2026-06-15 — "bolder" pass):** the hero plays ONE sequenced reveal the first time it scrolls into view — the red proof-strikes wipe in left→right (`scaleX` 0→1, `transform-origin` left, staggered ~80ms), then the cyan stamp lands. This is the correction happening live: markdown comes in wrong, gets marked, leaves stamped. Plays exactly once, never loops, never replays. Per-step timing stays ≤200ms; the full orchestration may total ~600ms. Strikes wipe via `scaleX` (a pen stroke), never a fade. Implemented with native CSS keyframes + one `IntersectionObserver` trigger — **no animation library** (penman's page stays dependency-free, as the pitch demands).
- Never: fades as primary transition, bounce/elastic/spring, ambient/looping motion, parallax, scroll-jacking — all still hold. The choreography is one-shot and transform-only; relaxing the law for it does NOT open the door to any of these.
- prefers-reduced-motion: all transforms removed, state changes instant — strikes render fully drawn, stamp visible, no sequence

## Page structure (information design — first-class, foundations §3)
Problem-first narrative (rev 2026-06-06): the gap between Claude's markdown and where it has to land.
1. **Hero — the problem (outcome-first / spine A)** — "Claude writes it clean. You paste it into Teams. Your team sees a mess." → "Penman bridges the gap." The frame is the handoff to a human reader, not the converter mechanics. Two sheets (signature move), CTAs
2. **Numbers strip** — oversized Fraunces numerals, reader-framed (not implementation brag): 13 places your reader is · 1 thing you do · 0 cleanup after paste. (The terminal target is deliberately excluded from "13 places your reader is" — the terminal is where the author pastes, not a reader surface — so the count stays 13.)
3. **The platforms** — the brag table: every platform's real recipe (slack · Lato · 15px …), 1px ruled, mono values. 13 *styled* platforms are tabled; the terminal target has no typographic recipe (it is plain text), so it is named in the lede, not given a row.
4. **The hard part** — per-target rendering, three modes (terminal added 2026-06-22): chat strips structure (bold + `<br>` + `<pre>` tables), document/email get full inline-styled HTML, and the **terminal** target strips everything to a runnable plain-text command (smart code-block extraction, trailing newline trimmed so the last line waits for Enter). Slack strips your HTML; penman renders differently per target. The terminal panel stays inside the paper metaphor — ink-on-paper mono, never retro-terminal cosplay (see Never).
5. **How it works** — Tokens → Render → Clipboard as three small sheets
6. **Customize** — `~/.penman.json5` token overrides, global and per-platform
7. **For Claude Code** — natural-language usage first ("pen that for teams"), the three MCP tools, marketplace install; slash command and CLI as the footnote
8. **Colophon footer** — set in its own recipe: "This page is set in Fraunces, Archivo & IBM Plex Mono." Requirements (macOS, Bun), repo link

## Never (this project's kill list)
- No gradients of any kind (incl. purple→blue AI default)
- No border radius — the "mixed radius" neo-brutalist variant is banned here
- No blurred shadows, no glassmorphism, no translucency
- No pure #000 or #fff — ink is #312d2c, paper is #fdfcfc
- Cyan never appears outside the after/fixed stamp and success states; no cyan-on-dark dashboard look
- Red never floods: no red sections, no red body text, no red backgrounds larger than a stamp/strike
- No bounce/spring easing, even on interaction — snap translates only
- No card-everything: bordered sheets only where the paper metaphor applies (demos, steps); prose sections stay open on the page
- No Inter/Roboto/Open Sans display duty
- No retro-terminal cosplay: no scanlines, no blinking cursors, no green-on-black
- No emoji as decoration
